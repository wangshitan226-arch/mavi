# 学迹

> 高中数学教师个人教学知识资产工作台  
> —— 让真实教学过程可记录、可分析、可复用、可交代

---

## 项目背景

在两段家教背景和一段机构背景的数学教育经历中，我发现现有的大学生家教存在以下问题

- 家教/机构教学效果**不透明**，家长和学生往往花钱了但不知道到底进步了什么，老师也难以用数据有交代。

- 家教/机构的老师入门门槛低，教学多**不成体系**，质量参差不齐，很容易产生**知识漂移**

本项目希望通过一个从老师端作为切入点的系统的实现，来解决上述问题

---

## 项目目标

- 实现教学过程可视化，向家长证明孩子真的进步了

- 以最低的门槛，实现让老师上手就有成体系的教学路线，并配以可视化教学工具，助力教学

- 不断地回归教材，不断地丰盈档案，保证知识不漂移，实现学习过程资产化

---

## 当前版本边界与取舍原则

我选择了一个政策友好、风险可控、但仍保留核心灵魂的版本，主要取舍如下：

- 完全教师端驱动，不做任何学生端主动使用界面

- 所有涉及学生的记录均为教师手动录入/整理，匿名化处理

- 不做任何形式的家长端自动推送、实时查看或云端学生数据共享

- 数据控制权完全属于教师：可随时导出、删除、不留云端学生可追溯身份信息

- 优先级排序：教师“可解释性与可交代性” > 使用体验 > 自动化程度 > 功能广度

- 当前阶段明确不追求“大而全”，而是“小而深”——先把教材+错因+可视化闭环跑通

## 项目功能

**一句话：** 系统以教材为知识骨架，持续将题目、错因和学习心得结构化地“挂回”到教材对应位置，从而让所有学习行为最终回归教材体系，避免知识漂移。

- [ ] 教材基座 + 动态知识结构树

> 教师上传教材 PDF 后，系统自动解析教材结构，
> 生成以“章节—小节—知识点”为骨架的动态知识结构树。

- [ ] 错因资产引擎 + 演化路径

>后续教师上传题目（例题 / 错题）及对应的典型错因，
>系统将题目涉及的知识点和错因，
>映射并绑定到教材结构中的对应节点。

- [ ] 可视化讲解资产库（Manim集成）

>使用manim工具实现可视化，助力老师讲解。

- [ ] 班级/学生匿名档案 + 效果趋势可视化

>所有围绕题目的理解、易错点、总结性说明，
>都作为“学习资产”持续沉淀在教材结构中，
>既可在教材旁回溯查看，
>也可独立汇总为学生的匿名学习档案。

- [ ] 整卷/阶段性诊断报告导出

- [ ] 个性化教学调整建议

- [ ] 长期成长档案导出

## 短期明确目标与里程碑

- 第1-2周
  完成教材 PDF 解析 + 章节结构树构建 + 基础知识节点挂载 demo  
  实现题目（文字/图片）上传 → 知识点&易错点初步结构化拆解

- 第3-4周
  完成 Manim 动态讲解一键生成与绑定功能  
  实现学生匿名档案 + 知识节点掌握趋势基础可视化

- 年前
  寒假小规模内测（目标：2~4位老师，6~12名学生）  
  收集第一轮真实使用反馈

- 年前
  根据内测反馈迭代，完成阶段性诊断报告（家长友好版）导出功能

- 年后
  整理项目文档、界面截图、核心数据模型，提交软件著作权申请

## 技术栈初步想法与主要技术挑战（随时讨论）

**倾向技术路线**

- 后端：Python + FastAPI + Pydantic

- 数据库：PostgreSQL（结构化数据） + MinIO/本地文件存储（PDF、图片、Manim输出）

- 文档解析：pdfplumber / PyMuPDF + 简单规则 + LLM辅助章节标题提取 / 自研 NLP 模块（知识点抽取）

- 题目解析与知识抽取：目前倾向直接调用大模型 + 强提示+规则后处理

- 可视化动画生成：Manim（社区版），初期预渲染 mp4，后期考虑 WebGL/Three.js 优化体验

- 前端：React + Vite + shadcn-ui / Ant Design

- 后续可能引入：轻量向量数据库（用于题目/易错点语义匹配）

**已知主要技术挑战**

1. 教材 PDF 自动章节层级准确提取（不同版本教材排版差异大）

2. 手写/印刷错题图片 → 结构化题目+知识点+易错类型（识别+语义理解双重难度）

3. Manim 渲染速度与教师端集成体验

4. 在完全教师控制、不存可追溯学生身份信息前提下，实现长期趋势追踪

5. 知识点与易错点的“可复用性”设计
